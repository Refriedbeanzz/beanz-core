## Session Commands

### Beginning of Session
When the user says "beginning of session" or "start of session":
1. Read DEVLOG.md to see what happened last session
2. Read PLAN.md to understand the current plan and priorities
3. Read ROADMAP.md to understand the broader goals
4. Read the Notion page above and note "Where We Left Off" and "Next Step"
5. Give a 3-5 sentence briefing: what state the project is in, what we were working on, and the recommended first thing to tackle

### End of Session
When the user says "end of session":
1. Append a dated entry to DEVLOG.md summarizing what was done this session — raw notes, decisions made, dead ends, context. Does not need to be polished.
2. Update CHANGELOG.md only if something shipped or a version changed — polished, user-facing entries (Added / Fixed / Changed / Breaking).
3. Update PLAN.md to reflect current state and next steps
4. Update ROADMAP.md if anything shipped, changed scope, or new features were planned
5. Update the Notion page: fill in "Where We Left Off", "Next Step", set "Last Session" to today, and add any new tasks to the linked Tasks database
